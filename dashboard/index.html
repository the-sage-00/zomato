✅ Benefits (Pros)

  • 28% Drop in P90 Error: Drastically reduces extreme wait times.
  • Zero Hardware Cost: Pure software solution using existing signals.
  • Self-Healing Data: Engine dynamically adapts to correcting merchant behavior over time.
  • Hotter Food & Happier Riders: 14% drop in food cooling time.

⚠️ Limitations (Cons)

  • Cold Start Problem: Needs ~200 orders for a new restaurant to build a stable profile. (Mitigated via Neighborhood peer-learning).
  • Compute Overhead: Maintaining sliding window analytics for 100K+ merchants requires a real-time stream processing architecture (like Kafka/Flink).
  • Extreme Edge Contexts: Massive weather events or road closures can temporarily skew the dwell time models until the system recalibrates.
